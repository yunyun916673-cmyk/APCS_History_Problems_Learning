字串操作
Description

給一個字串S，有三種字串操作，分別如下

1. 兩兩交換：將字串內相鄰兩個字元交換，例如字串 "apcsntnu" 先分組成 (ap)(cs)(nt)(nu)，將會交換為 (pa)(sc)(tn)(un)，即得到 "pasctnun"。
2. 兩兩排序：將字串內相鄰兩個字元按照字典序排序，字元的字典序為 a < b < ... < z，例如字串 "family" 先分組成 (fa)(mi)(ly)，將會交換成 (af)(im)(ly)，即得到 "afimly"。
3. 完美重排：假設字串長度為n，將字串內的字元編號為0,1,2,...,n−1，將字串分成兩半為0,1,⋯,n/2−1和n/2,n/2+1,⋯,n−1，並且組合成0,n/2,1,n/2+1,⋯,n/2−1,n−1。例如 "apcsntnu" 拆成 "apcs" 和 "ntnu"，然後交錯成 "anptcnsu"。

給定k個操作，請依序操作字串，輸出最後的字串結果。


Input

第一行輸入一個字串S(2≤|S|≤100)，字串長度保證為偶數。接下來一行輸入一個正整數k，接下來有k行，每一行有一個數字，種類為0,1,2。

(60 分):|S|≤10,k=1且保證操作為完美重排
(40 分): 無限制


Output

輸出操作完之後的字串內容。


Sample Input 1 

apcsntnu
1
2
Sample Output 1

anptcnsu
Sample Input 2 

facebook
4
2
0
2
1
Sample Output 2

bocfkoae
Hint

範例測資1如題序所述

範例測資2:
操作2: "facebook" 變為 "fbaocoek"。
操作0: "fbaocoek" 分組為 (fb)(ao)(co)(ek) 變為 (bf)(oa)(oc)(ke)，字串為 "bfoaocke"。
操作2: "bfoaocke" 變為 "bofcokae"。
操作1: "bofcokae" 分組為 (bo)(fc)(ok)(ae) 變為 (bo)(cf)(ko)(ae)，字串為 "bocfkoae"

# 字串操作

## 題目想法

一開始看到這一題的時候，我覺得這題應該需要針對三種不同的字串操作，分別寫出三個函式來處理字串 S。

因為題目給出的三種操作方式彼此獨立，如果全部寫在主函式中，不但會讓程式變得比較複雜，也不容易閱讀。因此我決定依照題目的三種指令，建立三個不同的函式，分別負責操作 0、操作 1 以及操作 2。

在開始撰寫程式時，我先建立字串 S，但是沒有急著在主函式中處理所有操作，而是先將三種字串變化的方法整理成函式，讓主程式只需要根據輸入的指令呼叫對應函式即可。

## 解題過程

第一個函式是處理「兩兩交換」。

題目要求將字串內相鄰的兩個字元分成一組，再將每一組中的兩個字元交換。

例如： apcsntnu

會分成： (ap)(cs)(nt)(nu)

再交換成： (pa)(sc)(tn)(un)

因此我利用迴圈走訪字串，每次增加 2，讓每一次迴圈都剛好處理一組字元，並利用 `swap()` 將兩個字元交換。

第二個函式是處理「兩兩排序」。

這個操作和第一個函式非常類似，同樣需要將字串中的字元兩兩分組。

不同的是，這次不是直接交換，而是需要比較兩個字元的字典序，讓字典序較小的字元排在前面。

因此我同樣使用每次增加 2 的迴圈方式，只是在交換之前增加判斷：

如果前面的字元字典序大於後面的字元，就交換兩者的位置。

第三個函式是處理「完美重排」。

這個操作需要先將字串切成左右兩半，再將左右兩邊的字元交錯組合。

例如： apcsntnu

拆成： apcs   ntnu

再依序組合：

anptcnsu

因此我先利用 `substr()` 將字串分成 `left` 和 `right` 兩部分。

接著建立一個新的空字串 `test`，利用迴圈依序加入：

- 左半部第 i 個字元
- 右半部第 i 個字元

最後再將原本的字串 S 更新成新的字串 `test`。

完成三個函式後，進入主函式。

首先輸入字串 S 和操作次數 K。

接著利用 K 次迴圈讀取每一次操作指令，根據輸入的數字：

- 0 呼叫兩兩交換函式
- 1 呼叫兩兩排序函式
- 2 呼叫完美重排函式

最後輸出經過所有操作後的字串 S。

## 解題方法思考

這題主要考驗的是如何將不同功能拆分成獨立的函式。

如果將三種操作全部寫在主函式中，雖然也可以完成題目，但是程式的結構會變得比較混亂。因此我選擇依照題目的操作種類建立函式，讓每個函式只負責一種字串處理方式。

這樣不只能讓程式更加簡潔，也能讓每個部分的邏輯更加清楚。

另外，這題也練習了許多字串操作，例如：

- 字串切割 `substr()`
- 字元交換 `swap()`
- 字元比較
- 建立新的字串

透過將不同功能分開處理，可以讓程式更容易修改與維護。

## 複雜度分析

假設字串長度為 n，操作次數為 k。

每一次操作都需要掃描一次字串：

- 兩兩交換：需要走訪整個字串，時間複雜度為 O(n)
- 兩兩排序：需要比較每一組字元，時間複雜度為 O(n)
- 完美重排：需要切割並重新建立字串，時間複雜度為 O(n)

因此總時間複雜度為： O(kn)

因為每一次操作都可能建立新的字串，所以額外使用的空間主要來自暫存字串： O(n)
