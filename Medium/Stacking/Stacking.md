# 物品堆疊 (Stacking)

題目描述

某個自動化系統中有一個存取物品的子系統，該系統是將 N 個物品堆在一個垂直的貨架上，每個物品各佔一層。

系統運作方式如下：

每次只會取用一個物品，取用時必須先將在其上方的物品貨架升高，取用後必須將該物品放回，然後將剛才升起的貨架降回原始位置，之後才會進行下一個物品的取用。

每一次升高某些物品所需要消耗的能量，是以這些物品的總重量計算。在此忽略貨架重量以及其他可能的消耗。

現在有 N 個物品，第 i 個物品的重量為 w(i)，需要取用的次數為 f(i)。

我們需要決定物品在貨架上的排列順序，使得總消耗能量最小。

輸入說明

第一行是一個整數 N，代表物品數量。

第二行有 N 個正整數：

w(1), w(2), ... , w(N)

代表每個物品的重量。

第三行有 N 個正整數：

f(1), f(2), ... , f(N)

代表每個物品需要被取用的次數。

限制：

1 <= w(i) <= 1000

1 <= f(i) <= 1000


輸出說明

輸出最小能量消耗值。

答案不會超過 63 位元正整數可以表示的範圍。

範例輸入 1

2
20 10
1 1

範例輸出 1

10


範例輸入 2

3
3 4 5
1 2 3

範例輸出 2

19

測資資訊

記憶體限制：64 MB

時間限制：1 秒


子題：

第 1 子題（10 分）

N = 2

f(1)=f(2)=1


第 2 子題（20 分）

N = 3


第 3 子題（45 分）

N <= 1000

每個物品取用次數 f(i)=1


第 4 子題（25 分）

N <= 100000


# 物品堆疊 (Stacking)

## 題目想法

一開始看到這一題的時候，我就知道不能直接使用窮舉，因為依照題目的限制來看，有很大的機率會造成 TLE。

因此我開始嘗試找尋題目中的規律。

我發現題目中的兩個變數 W 和 F 都會影響最後的能量消耗，而且兩者對結果造成的影響方向是一樣的。

如果一個物品的重量越小，放在下面時造成的影響就越小；而如果一個物品需要被取用的次數越少，它造成的額外消耗也會越少。

所以我開始思考兩個物品之間的排列關係。

後來我發現，當 W / F 越小時，這個物品越適合放在上面。因為放在上面的物品會影響較少次的升降，而將比例較大的物品放在下面，可以降低整體累積的重量。

因此我利用這個性質，透過排序找出最佳的排列順序。

## 解題過程

這一題因為最後答案可能很大，所以保險起見我使用 long long。

首先我利用 struct 定義一個新的資料型態 thing：

struct thing
{
    ll w, f;
};

用來同時存放每個物品的重量 W 以及取用次數 F。

進入主函式後，我先輸入物品數量 N，並建立一個存放 thing 的 vector：

vector<thing> all(n);

接著分別輸入每個物品的重量與取用次數。

因為後續需要利用 sort 對物品進行排序，所以我建立了一個比較函式 swapping。

在比較時，我原本需要比較：

W1 / F1 < W2 / F2

但是如果直接使用除法，可能會產生小數精度問題。

因此我改成使用乘法比較：

W1 * F2 < W2 * F1

避免浮點數造成錯誤。

排序完成後，我建立兩個變數：

weight： 代表目前已經累積的物品重量。

sum_： 代表最後的總能量消耗。


接著利用一次迴圈計算答案：

for (auto &test : all)
{
    sum_ += weight * test.f;
    weight += test.w;
}

每次加入新的物品時，先計算它需要承受下面所有物品重量造成的消耗，再將自己的重量加入累積重量中。

最後輸出 sum_。

## 解題方法思考

這一題最重要的地方不是實作，而是找到排序規律。

一開始我只知道不能暴力搜尋，但不知道應該如何排列物品。

後來觀察兩個物品交換位置時造成的影響，發現排列順序可以透過 W / F 來決定。

而在實作排序時，我沒有直接計算比例，而是利用交叉相乘比較，避免浮點數精度問題。

這也讓我了解到，在競賽題目中，很多看似需要模擬的問題，其實可以透過觀察兩個狀態交換時的差異，推導出排序規則。

## 複雜度分析

### 時間複雜度： O(N log N)

原因：

主要花費在 sort 排序。

排序需要比較 N 個物品，因此複雜度為 O(N log N)。

排序完成後，只需要使用一次迴圈計算答案，因此額外需要 O(N)。

所以總時間複雜度為：

O(N log N)

### 空間複雜度： O(N)

原因：

需要使用 vector<thing> 儲存所有物品資料。

## 學習心得

這一題主要訓練的不只是 vector 和 sort 的使用，更重要的是觀察問題現象並將它轉換成數學規律的能力。

以前遇到這類排列問題時，我常常會先想能不能模擬所有可能情況，但這題讓我了解到，如果限制較大，就必須嘗試找出物品之間交換時的關係。

透過這題，我學習到排序題的重點通常不是如何排序，而是如何找到「為什麼這樣排序會是最佳解」。

同時，我也更加熟悉了 C++ 中 struct、vector 以及自訂比較函式的使用方式。
