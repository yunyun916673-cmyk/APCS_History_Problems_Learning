# 棒球遊戲

## 題目描述

謙謙最近迷上棒球，他想自己寫一個簡化的遊戲計分程式。這會讀入隊中每位球員的打擊結果，然後計算出球隊得分。

這是個簡化版的模擬，假設擊球員打擊結果只有以下情況：

### 安打

- 1B：一壘打
- 2B：二壘打
- 3B：三壘打
- HR：全壘打

### 出局

- FO
- GO
- SO

簡化版規則如下：

1. 球場上有四個壘包，稱為本壘、一壘、二壘和三壘。

2. 站在本壘握著球棒打球的稱為擊球員，站在另外三個壘包的稱為跑壘員。

3. 當擊球員的打擊結果為安打時，場上球員（擊球員與跑壘員）可以移動；結果為出局時，跑壘員不動，擊球員離場換下一位擊球員。

4. 球隊總共有九位球員，依序排列。比賽開始由第 1 位開始打擊，當第 i 位球員打擊完畢後，由第 i+1 位球員擔任擊球員。當第九位球員完畢後，則輪回第一位球員。

5. 當打出 K 壘打時，場上球員（擊球員與跑壘員）會前進 K 個壘包。從本壘前進一個壘包會移動到一壘，接著是二壘、三壘，最後回到本壘。

6. 每位球員回到本壘時可得 1 分。

7. 每達到三個出局數時，一壘、二壘和三壘就會清空（跑壘員都得離開），重新開始。

請寫出具備這樣功能的程式，計算球隊總得分。

## 輸入說明

每組測試資料固定有十行。

前九行依照球員順序輸入。

每一行開始有一個正整數 a（1 ≤ a ≤ 5），代表該球員總共打了 a 次。

接下來有 a 個字串（均為兩個字元），依序代表每次打擊的結果。

資料之間均以一個空白字元隔開。

球員的打擊資訊不會有錯誤，也不會缺漏。

第十行有一個正整數 b（1 ≤ b ≤ 27），表示我們想要計算當總出局數累計到 b 時，該球隊的得分。

輸入的打擊資訊中至少包含 b 個出局。

## 輸出說明

輸出當總計第 b 個出局數發生時，球隊的總得分。

## 範例輸入 1

```text
5 1B 1B FO GO 1B
5 1B 2B FO FO SO
4 SO HR SO 1B
4 FO FO FO HR
4 1B 1B 1B 1B
4 GO GO 3B GO
4 1B GO GO SO
4 SO GO 2B 2B
```

棒球遊戲

題目想法

一開始看到這一題的時候，我覺得它本身並不困難，演算法也不會說很難想到。

我反而覺得這題最困難的地方，是需要維護大量的變數以及資料，

再加上要一直更新與掃描，讓整個程式需要考慮的事情變得很多。

我覺得這題主要是在考驗，當程式需要同時維護很多不同的資訊時，能不能快速把它們分類，

並依照它們各自的性質以及題目的要求去更新與掃描，而不是演算法本身有多困難。

因為這一題的 n 固定為 9，而且各項數值範圍都很小，所以我直接使用 int 就足夠了，不需要特別使用 long long。

解題過程

我先自訂了一個 Player 的資料型態，裡面存放一個 int 和一個 queue，

分別代表球員剩餘的打擊次數 times，以及每一次打擊的結果。

其實我後來覺得 times 在程式裡並沒有真的用到，只靠 queue 就可以完成整個模擬。

不過為了讓閱讀程式的人更容易理解目前球員還剩下多少次打擊，我還是決定把它保留下來，

雖然它不是必要的資訊，但我自己閱讀時會比較清楚。

接著建立題目要求的 b，再建立一個存放 Player 的 vector，並先把大小設成 9，因為題目固定只有九位球員。

接著我建立一個 bitset<3>，分別代表一壘、二壘以及三壘。如果某個位元是 0，

就表示該壘包目前沒有人；如果是 1，就表示該壘包有人。

進入 main 之後，我先輸入每一位球員的打擊次數，再把每一次打擊結果依序放進對應球員的 queue 中。

之後輸入 b，並初始化目前打擊者為 -1，代表目前還沒有人打擊，同時把目前得分初始化為 0，累積出局數也初始化為 0。

接著開始模擬整場比賽，直到總出局數達到 b 為止。

每一次迴圈先更新目前的打擊者，讓下一位球員上場，再取出他這一次打擊的結果。

如果這次不是安打，而是出局，那麼就把出局數加一。

如果累積出局數剛好是 3 的倍數，就代表半局結束，需要把三個壘包全部清空，再直接進入下一次迴圈。

如果是安打，我就先根據打擊結果決定這次需要前進幾個壘包，並用 thehit 紀錄。

接著我不是直接分別處理一壘打、二壘打、三壘打以及全壘打，

而是每次都讓所有跑壘員往前移動一個壘包，再重複這個動作直到前進 thehit 步為止。

第一次移動時，再把目前的打擊者放上一壘，之後就讓所有人一起持續往前推進。如果有人回到本壘，就把分數加一。

當總出局數達到 b 之後，直接輸出目前累積的得分即可。

解題方法思考

我一開始就在想，要怎麼表示目前三個壘包的狀態。

後來想到，其實三個壘包只有有人或沒有人兩種狀態，所以我直接使用 bitset<3> 去表示目前三個壘包是否有人，

這樣更新起來非常方便，也不用另外建立很多變數。

另外，我沒有分別去寫一壘打、二壘打、三壘打以及全壘打四種不同的更新方式，

而是把它們全部統一成重複前進一個壘包的操作。這樣整體程式雖然需要多跑幾次迴圈，

但因為題目的資料範圍非常小，所以效率完全不是問題，反而讓整體邏輯一致，也比較不容易漏掉某些情況。

## 複雜度分析

### 時間複雜度： O(n)

輸入所有打擊結果最多只有 45 筆，而模擬過程中，每一次打擊最多只需要前進 4 次，

每次再掃描三個壘包，因此整體操作次數都是固定範圍內，會隨著輸入資料量線性增加，因此時間複雜度為 O(n)。

### 空間複雜度： O(n)

主要額外使用了一個儲存所有球員打擊紀錄的 vector 以及 queue，

另外再使用一個 bitset 紀錄三個壘包的狀態，因此整體空間會隨著輸入資料量線性增加，所以空間複雜度為 O(n)。
