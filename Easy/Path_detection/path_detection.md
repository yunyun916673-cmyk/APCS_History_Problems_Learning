路徑偵測
Description

給一個二維平面，座標如同數學的二維座標(Ｙ正為北，Ｘ正為東)。起始位置在 (0, 0)，接下來會有n個座標，你需要按照這些座標點的順序移動，保證僅會垂直或水平方向上移動，不會斜向移動，且第一個點保證一定是Ｘ軸正的位置(初始方向向右)。

請輸出這條路徑中，左轉、右轉、迴轉的個數分別為多少。

子問題一 (60%) n=2
子問題二 (40%) n<=100


Input

第一行輸入一個正整數n，接下來有n行，每一行都有兩個正整數x,y。保證的是相鄰兩個點的座標差值不超過100。


Output

輸出三個正整數，分別代表左轉、右轉、迴轉的次數。


Sample Input 1 

2
2 0
2 1
Sample Output 1

1 0 0
Sample Input 2 

9  
4 0
4 9
4 8
4 10
4 2
4 3
6 3
6 10
6 9
Sample Output 2

2 1 5


# 路徑偵測

## 題目想法

一開始看到這個題目的時候，我愣了一下，因為比起常見的二維陣列表示方式（row 和 col），這題使用的是數學上的 x、y 座標平面。

不過仔細想過後，我發現兩者其實沒有太大的差別，只是座標方向的定義不同而已。由於題目會直接給每一次移動後的座標，因此我不需要真的建立整張地圖，也不用模擬每一步經過的格子。

我觀察到題目保證每次移動一定只會往垂直或水平方向移動，不會有斜向移動，因此我只需要比較新的座標與舊的座標之間的差異，就可以知道目前移動的方向。

另外，因為題目的限制很小，n 最大只有 100，所以不需要使用複雜的資料結構，直接使用 int 儲存座標即可。

## 解題過程

一開始，我先輸入 n，並初始化幾個需要維護的資訊：

- 目前的 x 座標 nowx
- 目前的 y 座標 nowy
- 目前面朝的方向 nowdir

我將方向按照順時針方式編碼：

- 0：向右（東）
- 1：向下（南）
- 2：向左（西）
- 3：向上（北）

這樣設定的原因是之後比較方向變化時，可以直接利用數字差值判斷轉向情況。

接著建立三個變數：

- right：右轉次數
- left：左轉次數
- turn：迴轉次數

然後進入 n 次迴圈，每次讀入新的座標 newx、newy。

接著利用目前座標和新的座標比較，判斷新的移動方向：

- 如果 x 座標沒有改變，代表垂直移動，再利用 y 座標判斷往上或往下。
- 如果 y 座標沒有改變，代表水平移動，再利用 x 座標判斷往左或往右。

判斷完成後更新 newdir。

接著比較： (newdir - nowdir + 4) % 4


得到方向改變的種類：

- 差值為 1：代表右轉
- 差值為 3：代表左轉
- 差值為 2：代表迴轉

根據結果增加對應的計數。

最後更新：

- nowx
- nowy
- nowdir

讓下一次迴圈可以繼續判斷。

當所有座標都處理完成後，輸出左轉、右轉以及迴轉的數量即可。

## 解題方法思考

這一題讓我注意到，看到二維平面問題時，不一定要直接建立整張地圖。

一般在處理二維問題時，常見的想法會是建立 row 和 col 的陣列，再利用掃描或搜尋來模擬整個過程。

但這題給出的資訊其實已經足夠，因為它直接提供每次移動後的位置。

因此，比起建立整個平面，我選擇直接比較前後兩個座標，取得移動方向，再透過目前面朝方向判斷轉向。

這樣不但程式更加簡潔，也避免了建立不必要的空間。

另外，這題也有一定程度上的演算法選擇。

雖然看起來像是在模擬路徑，但實際上並不需要真的模擬每一步，而是利用題目給予的限制，抓出需要維護的資訊。

## 複雜度分析

時間複雜度： O(n)
原因是每一個座標只需要讀取一次，並進行一次方向判斷，因此總共執行 n 次操作。

空間複雜度： O(1)

原因是程式只維護目前座標、方向以及三種轉向次數，沒有建立額外的陣列或資料結構。

## 學習心得

對我而言，這一題算是一題滿有趣的基礎題。

因為在寫程式時，學生通常會習慣使用 row 和 col 去建立二維平面，但這題則是使用平常在學校數學中接觸的 x、y 座標系統來進行模擬。

這讓我發現，二維問題不一定都需要透過矩陣或地圖處理，重點是在於分析題目真正需要哪些資訊。

這題也訓練了我判斷問題本質的能力。表面上它像是在追蹤一條路徑，但實際上只需要維護座標以及目前方向即可完成。

整體而言，這是一題讓人眼前一亮的基礎性題目，雖然難度不高，

但能幫助我練習如何從題目限制中選擇更適合的解法，而不是直接使用最直覺的方法。
