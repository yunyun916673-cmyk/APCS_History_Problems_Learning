# 遊戲選角

## 題目想法

一開始看到這一題的時候，我下意識想到的方法是在迴圈裡面維護兩個變數，分別代表目前能力值最大的角色以及第二大的角色。

每次輸入一個新的角色後，先計算它的能力值（攻擊力平方加上防禦力平方），再判斷它是否比目前第一大的角色還要大。

如果比第一大還大，那麼新的角色會變成第一大，而原本的第一大則會往後變成第二大。

如果沒有超過第一大，則繼續判斷它是否比第二大還大，如果是的話，就更新第二大的角色。

不過仔細想過後，我發現這種方法雖然可以解決問題，但是整體邏輯比較複雜，需要維護第一大與第二大的狀態，而且判斷流程較多，程式閱讀起來也沒有那麼直觀。

因此我開始思考是否能利用排序的方法來簡化問題。

一開始我想到使用 multiset，因為它本身會自動維持排序，可以在插入角色時保持能力值的順序。

但是實際寫法上，我發現 multiset 並不能直接利用索引取值。如果要取得第二大的元素，需要先建立迭代器：

auto it = ch.end();
--it;
--it;

再透過迭代器取出資料。

雖然可以完成，但我覺得這樣的寫法反而讓程式邏輯變得比較不自然。

後來查詢資料後，我發現使用 vector 儲存所有角色，最後再利用 sort 排序會更加適合。

雖然 multiset 與 vector + sort 的時間複雜度都是： O(nlogn)

但 multiset 每次插入資料時都需要重新排序，而 vector+sort 則是全部輸入完成後再一次排序，實際執行時常數較小。

因此最後選擇使用 vector+sort 的方式完成此題。

## 解題過程

這題的能力值範圍在題目限制內並不大，因此int已經足夠，不需要使用 long long。

首先輸入角色數量 n，建立一個：

vector<tuple<int,int,int>> ch(n);

其中每個 tuple 儲存三個整數：

能力值
攻擊力
防禦力

接著進入迴圈讀取每個角色的資料。

輸入：

int a,d;

分別代表攻擊力與防禦力。

計算能力值：

int ab=a*a+d*d;

因為之後需要依照能力值排序，所以我將資料以： (ab,a,d) 的順序存入 tuple。

也就是：

ch[i]={ab,a,d};

如此一來，在排序時，tuple 會優先比較第一個元素，也就是能力值。

全部角色輸入完成後：

sort(ch.begin(),ch.end());

將角色依照能力值由小到大排序。

排序完成後，由於我已經知道角色數量為 n，所以不需要額外將排序改成由大到小。

排序後：

第二大能力值的位置為 ch[n-2]

因此直接取出：

get<1>(ch[n-2])
get<2>(ch[n-2])

即可得到第二大角色的攻擊力與防禦力。

## 解題方法思考

這一題其實有很多種方法可以完成。

第一種方法是我最開始想到的維護法：

使用兩個變數分別保存第一大與第二大的能力值。

這種方法不需要排序，因此時間複雜度可以降低到： O(n)

而且如果只需要保存攻擊力與防禦力，也可以直接使用變數或 pair 儲存。

這種解法主要考驗的是：

判斷條件設計能力
更新狀態的能力
對變數關係的掌握

而不是排序能力。

而我最後選擇的 vector+sort 方法，主要考驗的是資料存儲與 STL 的使用能力。

例如：

-  tuple 儲存多項資料
- 使用 vector 儲存資料
- 使用 sort 進行排序
- 使用 get 取出 tuple 中的指定資料

雖然這個方法的時間複雜度比維護第一大、第二大的方法高，但是因為本題的資料範圍很小，所以沒有必要過度追求最低複雜度。

在這種情況下，選擇一個邏輯清楚、容易理解的寫法，可能比單純追求效率更重要。

複雜度分析
本題使用的方法：vector + sort

時間複雜度：

O(nlogn)

原因：

讀取所有角色需要一次迴圈：
O(n)
最後使用 sort 排序：
O(nlogn)

因此主要時間花費在排序。

空間複雜度：

O(n)

原因：

需要使用 vector 儲存所有角色資料。

其中 tuple 額外存放三個整數，但常數係數可以省略，因此仍記為：

O(n)

如果使用第一大、第二大維護法

時間複雜度：

O(n)

原因：

只需要掃描一次所有角色，每次比較並更新答案。

空間複雜度：

O(1)

原因：

只需要保存目前最大的兩個角色資訊，不需要儲存所有資料。

學習心得

這一題雖然是一題基礎題，但我覺得它很適合作為資料處理能力的練習。

透過這題，我練習了如何將一個問題轉換成適合使用排序解決的形式。

一開始我想到的是維護第一大與第二大的方法，這是一種更偏向直接模擬的思考方式。

但是經過比較後，我發現有時候不一定需要自己維護複雜的狀態，也可以利用已經存在的工具，例如 sort，讓程式邏輯更加簡單。

同時，這題也讓我更熟悉：

vector
tuple
sort
get

這些 C++ 常用工具。

整體來說，這是一題測驗基礎程式能力的題目，不同方法都有自己的優點。

有些方法追求更低的時間複雜度，有些方法則追求程式更容易理解。

在實際解題時，應該根據題目限制選擇適合的方法，而不是永遠只追求最低複雜度。
