機械鼠
Description

有n個位置上有食物，另外有一隻老鼠一開始位於位置x。

老鼠在開始覓食前要選擇今天要往左邊或往右移動去尋找食物，經過食物時可以停下來吃食物，吃完後可以選擇繼續往相同方向移動，或者是結束今天的覓食。

請問老鼠最多能吃到多少個食物，以及最後停下來吃食物的位置。


Input

第一行包含兩個整數：x和n，以空格分隔。x代表老鼠的初始位置，n代表食物的數量。

第二行包含n個整數，以空格分隔，表示每個食物的位置，且不會與老鼠位置重疊。

所有測試資料皆保證3≤n≤20且n是奇數，老鼠與食物位置範圍均為−100到100。

子題分數：

60%：滿足n=3。
40%：一般情況。

Output

請輸出兩個整數，分別代表最多能吃到的食物數目和最後一個吃的食物停下的位置。


Sample Input 1 

10 3
1 5 13
Sample Output 1

2 1
Sample Input 2 

10 9
-1 13 12 16 100 -9 7 8 25
Sample Output 2

5 100

# 機械鼠

## 題目想法

一開始看到這個題目的時候，我先想到的是讓老鼠分別往左邊以及右邊移動搜尋，然後判斷每一種方向可以吃到多少食物。

但是仔細觀察題目後，我發現其實不需要真的模擬老鼠移動的過程。因為食物的位置以及數量都是固定的，而老鼠一開始只需要選擇往左或往右其中一個方向前進。

因此，我只需要計算：

- 老鼠往左走時，可以經過多少個食物。
- 老鼠往右走時，可以經過多少個食物。

再比較兩邊哪一邊可以吃到比較多食物即可。

在思考過程中，我發現這題的重點其實不是模擬，而是如何快速找到某個位置在排序後資料中的位置，因此想到利用 lower_bound 與 upper_bound 來計算左右兩側的食物數量。

## 解題過程

首先依照題目要求輸入老鼠的位置 x 以及食物數量 n。

接著建立一個 vector foods，用來儲存所有食物的位置，並將每一個食物輸入到 foods 中。

由於後續需要快速找到食物與老鼠位置的關係，因此先使用 sort 將 foods 由小到大排序。

排序完成後：

- 使用 lower_bound 找到第一個大於等於 x 的位置。
- 使用 upper_bound 找到第一個大於 x 的位置。

透過這兩個位置，可以計算：

- 老鼠往左移動時可以吃到的食物數量。
- 老鼠往右移動時可以吃到的食物數量。

最後比較 left 和 right：

- 如果往右可以吃比較多食物，就輸出 right 以及最右邊的食物位置。
- 否則輸出 left 以及最左邊的食物位置。

另外，由於題目的資料範圍只有 -100 到 100，因此使用 int 就已經足夠，不需要使用 long long。

---

## 解題方法思考

一開始我想到的方法是直接搜尋老鼠移動的過程，但後來發現這樣其實沒有必要。

因為老鼠選擇方向後，移動路徑是固定的，不存在中途改變方向的情況，所以只需要知道該方向上有多少食物即可。

我原本也有想過使用 find 直接搜尋符合條件的位置，這種方法比較直觀，而且在本題的資料範圍內也不會造成 TLE。

但是後來考慮到：

- find 需要逐一遍歷資料。
- lower_bound 和 upper_bound 在排序後可以使用二分搜尋。

因此選擇使用 lower_bound 與 upper_bound，除了時間複雜度較低之外，程式碼也更加簡潔。

這題主要讓我練習到如何利用排序後的資料特性，快速取得資料的位置，而不是直接進行全部搜尋。

## 複雜度分析

時間複雜度：

O(n log n)

原因：

首先需要使用 sort 對 n 個食物位置排序，時間複雜度為 O(n*log n)。

排序後使用 lower_bound 與 upper_bound 尋找位置，兩者皆為二分搜尋，時間複雜度為 O(log n)。

因此整體時間複雜度由排序主導，為 O(n*log n)。

空間複雜度：

O(n)

原因：

需要使用 vector foods 儲存所有 n 個食物的位置。

## 學習心得

這一題算是一個比較基礎的資料位置應用題。

其中包含：

- 資料儲存
- 排序
- 二分搜尋
- 條件判斷

雖然題目本身不難，但我覺得它比較特別的是，它沒有要求直接模擬，而是需要觀察題目的特性，把問題轉換成尋找資料位置。

在 APCS 簡單程度的題目中，這類利用排序後的位置關係來解題的方法相對比較少見，因此這題讓我更加熟悉 lower_bound 與 upper_bound 的應用。

也讓我了解到，遇到看似需要模擬的題目時，不一定真的要按照題目流程一步一步執行，有時候分析資料的特性，找到更直接的方法會更加有效率。
