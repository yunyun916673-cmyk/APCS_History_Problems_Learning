# 邏輯運算子
題目敘述
給定三個整數 a、b、t。
其中 a 和 b 為非負整數，但在進行運算時，不直接使用它們原本的數值，而是先將它們轉換成邏輯值：
0 代表 false
非 0 代表 true
接著分別對 a 與 b 進行三種邏輯運算：
AND
OR
XOR
如果某一種運算得到的結果與 t 相同，就輸出該運算名稱。
若有多種運算符合條件，必須依照 AND、OR、XOR 的順序輸出，每種運算各占一行。
如果三種運算都無法得到 t，則輸出 IMPOSSIBLE。
輸入格式
輸入只有一行，包含三個整數：
a：第一個非負整數
b：第二個非負整數
t：期望得到的邏輯運算結果，只會是 0 或 1
三個整數之間以空白隔開。
輸出格式
依序檢查 AND、OR、XOR 三種邏輯運算。
若某種運算結果等於 t，就輸出其名稱。
如果沒有任何運算符合條件，輸出
IMPOSSIBLE
每個答案皆需獨立占一行。
題目重點
先將 a、b 轉換為邏輯值，而不是直接使用原本的整數。
0 視為 false，其餘非 0 的數皆視為 true。
運算結果只可能為 0 或 1。
若有多個符合答案，輸出順序固定為 AND、OR、XOR。
若完全沒有符合的運算，輸出 IMPOSSIBLE。

# 邏輯運算子

## 題目想法

一開始我看到這個題目的時候，其實有點不太熟悉，因為那時候我才剛學到 AND、OR 和 XOR 這三種邏輯運算，知道它們的概念，但還沒有真正寫過相關的題目，所以對它們還不是很熟悉。

這一題的資料範圍很小，因此我直接使用 int 就可以了。由於題目的運算結果只會是 0 或 1，所以我先把所有大於 1 的整數都轉換成 1，方便後續直接進行判斷，不需要再額外處理其他數值的情況。

## 解題過程

我先建立變數 a、b、t 來儲存輸入，再建立一個布林變數 pass，紀錄是否有任何一種運算符合題目要求，方便最後判斷要不要輸出 IMPOSSIBLE。

接著先將 a 和 b 大於 1 的數值都轉換成 1，因為題目會將非 0 的數都視為 true，而運算結果也只會是 0 或 1，這樣後續判斷會比較方便。

完成轉換後，我依序判斷 AND、OR、XOR 三種運算的結果是否等於 t。如果符合，就依照題目要求輸出對應的運算名稱，並將 pass 設為 true。

最後，如果三種運算都沒有符合，就代表沒有任何答案，因此輸出 IMPOSSIBLE。

## 解題方法思考

這一題不需要使用任何複雜的演算法，只要依照題目的規則模擬三種邏輯運算即可。

重點在於先理解題目是將輸入轉換成邏輯值後再進行運算，而不是直接比較原本的整數。理解這個規則之後，整個程式的實作就變得十分直接，只需要依序檢查三種運算即可。
