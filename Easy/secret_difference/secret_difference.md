秘密差

內容
問題描述

將一個十進位正整數的奇數位數的和稱為A ，偶數位數的和稱為B，則A與B的絕對差值 |A －B| 稱為這個正整數的秘密差。

例如： 263541 的奇數位和 A = 6+5+1 =12，偶數位的和 B = 2+3+4 = 9 ，所以 263541 的秘密差是 |12 －9|= 3 。

給定一個 十進位正整數 X，請找出 X的秘密差。

原題pdf檔

輸入說明
輸入為一行含有一個十進位表示法的正整數X，之後是一個換行字元。

輸出說明
請輸出 X的秘密差 Y(以十進位表示法輸出 )，以換行字元結尾 。

範例輸入 #1
263541
範例輸出 #1
3
範例輸入 #2
131
範例輸出 #2
1

# 秘密差

## 題目想法

一開始看到這個題目的時候，我想到的是需要分別計算數字每一位的位置，判斷它是奇數位還是偶數位，然後分別加入 A 和 B。

因為題目給的是十進位正整數，所以如果直接使用整數來處理，需要不斷利用除法和取餘數取得每一位數。不僅這樣會比較麻煩，而且如果數字範圍變大，也可能受到整數大小限制。

我發現這一題其實不需要真的把數字當成數值計算，而是只需要讀取每一個位數。因此我選擇直接將輸入的數字轉成字串處理，利用字串的位置來判斷目前是哪一位，再分別更新 A 和 B。


## 解題過程

我先初始化兩個變數 A 和 B，分別用來儲存奇數位數字和偶數位數字的總和。

接著輸入數字 X，並將它以字串形式儲存，這樣就可以直接存取每一個字元。

開始掃描字串中的每個位置：

如果目前的位置是奇數位，就將這個數字加入 A。

如果目前的位置是偶數位，就將這個數字加入 B。

因為字串的索引從 0 開始，所以判斷時使用 i+1 轉換成題目中的位數。

最後計算 A 和 B 的絕對差值，輸出答案。


## 解題方法思考

這一題主要是在練習如何處理位數較大的數字。

一開始也可以考慮利用除法和取餘數的方式取得每一位，但這種方式需要額外處理位數順序，而且當數字很大時會受到資料型態限制。

因此我選擇使用字串處理數字，因為字串可以直接存取每一位，也不用考慮數字大小限制。

這種將大數轉換成字串處理的方法，在之後遇到需要處理非常大的整數時也會比較方便。


## 複雜度分析

時間複雜度：

O(N)

原因：

需要將輸入數字的每一位掃描一次，因此時間會隨著數字長度 N 增加。


空間複雜度：

O(N)

原因：

需要使用字串儲存輸入的 N 位數字。


## 學習心得

這一題雖然是一份基礎題目，但讓我練習了遇到大數字時的處理方式。

遇到數字問題時，容易直接想到使用 int 或 long long 計算，但這題讓我發現，有些題目其實不需要真的進行數值運算，只需要處理每一位數的位置。

將數字轉成字串後，可以更方便地存取每一位，也避免受到資料範圍限制。

這也我告訴我，遇到數字大小可能超過限制的問題時，可以思考是否能轉換成字串，利用其他方式處理。
