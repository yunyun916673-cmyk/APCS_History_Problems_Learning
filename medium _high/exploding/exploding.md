連鎖反應
Description

在一個 M x N 的網格中，每一格可能是石頭或是炸彈，具體包含以下數字：

-1表示石頭，無法被炸彈引爆，炸彈震波無法通過該格傳遞。
-2表示初始炸彈的起始點，可以設定爆炸半徑。
其他整數表示該位置存在爆炸半徑為該數值的炸彈。
炸彈爆炸時，會影響周圍一定範圍內的格子。炸彈會以該格為中心擴散，若一個炸彈的爆炸半徑為v，則會將從該格開始沿著上、下、左或右走v步內可以走到的格子都引爆。若某一顆炸彈引爆的範圍中涵蓋到尚未引爆的炸彈，則會引發鏈鎖反應將尚未引爆的炸彈引爆，並依循著相同的規則發生鏈鎖反應。

目標是找出初始炸彈所需設置的最小爆炸半徑，使得至少有q個格子被引爆。


Input

第一行有三個正整數M,N,q(2≤M,N≤500,1<q≤M×N)，接下來有M行，每一行有N個數字。保證炸彈半徑為正數的數量最多只有1500個，且在場上的炸彈半徑不超過30。

保證一定存在一個初始炸彈的爆炸半徑使得引爆的格子數量至少q。

(20%):1≤M,N≤100，炸彈半徑為正的數量不超過100且場上沒有任何石頭。
(40%):1≤M,N≤200，炸彈半徑為正的數量不超過200，場上的炸彈半徑不超過20。
(40%): 無限制


Output

輸出至少需要設置爆炸半徑多大的炸彈，才能使得至少有q個格子被引爆。


Sample Input 1 

3 5 10
0 2 0 0 1
0 0 0 1 0
-2 0 0 0 0
Sample Output 1

3
Sample Input 2 

4 6 10
0 0 -1 -1 -1 0
0 0 -1 1 -1 2
0 -1 0 -1 0 0
2 -2 0 0 0 -1
Sample Output 2

4

# 連鎖反應

## 題目想法

一開始看到這一題的時候，我覺得它邏輯上其實不會到很困難，但這也是我第一次挑戰 APCS 歷屆試題中的中高級題目。真正開始寫的時候，我才發現事情沒有想像中那麼簡單，不管怎麼嘗試都做不出來。後來查了很多資料，也問了不少相關的問題，再加上自己做了大量的 BFS 練習之後，才終於能夠靠自己把這一題完成。

一開始分析題目的時候，我發現炸彈引爆後還會引爆其他炸彈，因此不能只考慮初始炸彈的爆炸範圍，而是要把整個連鎖反應都模擬出來。另外，題目要求的是最小的初始爆炸半徑，我發現只要半徑越大，引爆的格子數就只會越多，不會變少，因此答案具有單調性，可以利用二分搜尋去找最小的答案，而每一次判斷則利用 BFS 去模擬整個爆炸過程。

---

## 解題過程

這一題我採用了雙層 BFS 的結構。

外層的 BFS 用來處理炸彈之間的連鎖反應，每當有新的炸彈被引爆，就把它加入佇列，等待之後繼續向外爆炸；內層的 BFS 則負責模擬單一顆炸彈的爆炸範圍，利用 BFS 計算在爆炸半徑內可以到達哪些格子，同時避開石頭，因為石頭無法被引爆，震波也不能穿過石頭。

一開始我先建立題目所需要的變數 `m`、`n`、`q`，以及地圖 `grid`。在讀入地圖的同時，我會找到初始炸彈的位置，記錄在 `sr` 和 `sc` 中。

接著我建立 `BFS` 函式，函式中的參數 `org` 代表初始炸彈目前測試的爆炸半徑。函式內先建立 `visited`，用來記錄哪些格子已經被引爆，避免重複計算；再建立 `temp` 佇列，用來存放等待引爆的炸彈，並先把起點放入佇列中。同時利用 `bombed` 記錄目前總共引爆了多少格。

每次從 `temp` 取出一顆炸彈後，我會先決定它的爆炸半徑。如果是初始炸彈，就使用目前二分搜尋測試的半徑；如果是地圖上的炸彈，就使用它本身的爆炸半徑。

接著開始內層 BFS。我建立 `dist` 紀錄每個格子距離目前爆炸中心的步數，再利用 `spread` 佇列向四個方向擴散。每次擴散時，都會先判斷是否超出地圖、是否撞到石頭，或是否已經拜訪過；如果都沒有，再計算新的距離 `theDist`，只有在距離沒有超過爆炸半徑時才會繼續擴散。

當新的格子第一次被引爆時，我會更新 `visited`，並增加 `bombed` 的數量。如果這個格子本身也是炸彈，就把它加入外層 BFS 的佇列，之後再由它繼續引發新的爆炸。整個模擬完成後，再判斷引爆的格子數是否已經大於等於 `q`，作為二分搜尋判斷的依據。

最後回到主函式，利用二分搜尋找出最小的初始爆炸半徑，只要目前半徑能夠引爆至少 `q` 個格子，就繼續往更小的答案搜尋，否則就增加半徑，最後得到符合題意的最小解。

## 解題方法思考

一開始我一直在思考要如何處理炸彈之間的連鎖反應，後來發現其實可以把問題拆成兩個部分。

第一個部分是炸彈之間的連鎖反應，因此利用一層 BFS 去管理哪些炸彈還需要爆炸；第二個部分則是單一顆炸彈的爆炸範圍，因此再利用另一層 BFS 去模擬爆炸向四周擴散的過程。將問題拆開之後，整體邏輯就變得清楚許多。

另外，我也注意到初始炸彈的爆炸半徑越大，引爆的格子數一定不會變少，因此答案具有單調性。利用二分搜尋搭配雙層 BFS，就能有效率地找到符合條件的最小半徑，而不用從小到大一個一個去測試。

## 複雜度分析

**時間複雜度：**

**O(log(M+N) × K × R²)**

其中 `K` 為場上炸彈的數量，`R` 為炸彈的最大爆炸半徑。由於題目限制 `R ≤ 30`，可視為常數，因此整體效率足以通過題目的限制。

**空間複雜度：**

**O(M × N)**

主要來自 `visited`、`dist` 等二維陣列，以及 BFS 使用的佇列。
