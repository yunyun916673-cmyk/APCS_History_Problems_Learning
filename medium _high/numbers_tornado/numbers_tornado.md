# 數字龍捲風

## 題目想法

一開始看到這個題目的時候，我就覺得這一題應該會很麻煩，因為題目要求從正中間開始走，代表我需要不斷判斷什麼時候要轉彎，以及目前走到哪裡。

一開始我嘗試按照題目的方向去模擬，但是在寫的過程中，發現需要維護的資訊越來越多，例如目前方向、走了幾步、什麼時候需要改變方向等等，整個判斷變得非常複雜。

後來我突然想到，既然最後只是需要輸出走訪順序，那我是否可以反過來思考，直接從外面開始走，最後再把答案反轉輸出。

因為從外面開始走的話，我只需要判斷下一步是否超出邊界，或是是否走過的位置即可。而且利用螺旋這種近似於轉輪的特性，當碰到邊界或已經走過的位置時，就代表需要轉向。

因此我選擇改成從外圍開始模擬，最後再將結果反轉。


## 解題過程

我先輸入 N 並建立整個地圖，使用二維陣列 grid 儲存每個位置的數字。

另外建立一個二維布林陣列 visited，用來記錄哪些位置已經走訪過，避免在旋轉過程中重複經過。

接著輸入題目給出的起始方向。

因為我的走法是從外面往中心走，所以需要將原本的方向反過來，並根據原本的方向決定開始的位置。

例如：

- 第一步向左，代表最後會從左下角結束，因此我從左下角開始往右走。
- 第一歩向上，代表最後會從左上角結束，因此我從左上角開始往下走。

完成起點與初始方向設定後，就開始模擬走訪。

每一次移動時：

1. 先判斷下一個位置是否超出地圖範圍。
2. 判斷下一個位置是否已經走訪過。
3. 如果不能前進，就改變方向。
4. 如果可以前進，就移動到新的位置，並將數值加入答案。

因為題目的 N 一定是奇數，所以中心的位置一定是：

n / 2

當走到中心後，代表整個反向走訪完成。

最後將儲存的數字順序反轉，就可以得到題目要求的從中心開始的龍捲風順序。


## 解題方法思考

這一題真正困難的地方不是實作，而是在開始寫程式之前思考要使用什麼方式模擬。

如果直接按照題目的描述，從中心開始往外旋轉，需要額外判斷目前走了多少步，以及什麼時候需要改變方向，整個流程會變得比較複雜。

但是換一個方向思考後，從外圍開始走會簡單很多。

因為只需要判斷：

- 是否超出邊界
- 是否走過

而不需要另外計算每一段需要走幾格。

這讓整個模擬過程變得更加直覺。

這題讓我了解到，有些題目不一定需要按照題目給出的方式直接模擬，可以嘗試改變問題的方向，找到更容易處理的方法。


## 複雜度分析

時間複雜度：

O(N²)

原因：

輸入地圖需要讀取 N² 個數值，而走訪過程中每個格子也只會經過一次，因此整體時間複雜度為 O(N²)。

空間複雜度：

O(N²)

原因：

需要使用二維陣列 grid 儲存地圖，以及二維陣列 visited 記錄走訪狀態，兩者大小皆為 N²。


## 學習心得

這一題雖然實作本身不算困難，但是它訓練了我改變思考方向的能力。

一開始我一直想按照題目描述的方法直接模擬，結果發現需要處理非常多細節。後來換個方向思考，改成從外面往中心走，整個問題突然簡單很多。

我學到遇到模擬題時，不一定要完全照著題目敘述的方向做，可以先思考是否能轉換問題的角度。

有時候改變模擬方向，就能讓原本複雜的判斷變成很簡單的條件判斷。
