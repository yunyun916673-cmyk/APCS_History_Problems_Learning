重組問題
Description

對於一個長度為n的陣列a1,a2,⋯,an，已知a1為0，定義Δ為陣列中任兩個數字的絕對值差所組成的一群數字。

給定Δ的內容，輸出可以生成該Δ的最小與最大字典序陣列。

舉例來說，如果n=3且Δ=(3,4,7)，距離最大的是7，因為最大距離必定是在最小與最大兩數之間，

我們可以推論原序列的最大的數字是7。接著，目前已知原序列有(0,7)，而距離序列扣除7之後剩下(3,4)，

我們要決定中間點的位置。此時最大的距離是4，這個距離必然是該未知點與「最小的0」或者與「最大的7」之間的距離，

所以這個點可能是4或者3，在檢驗距離序列後，發現兩者皆可能，所以原序列可能是(0,3,7)或者(0,4,7)，

其中字典序最小的是(0,3,7)，最大的是(0,4,7)。


Input

第一行輸入一個正整數n(1≤n≤25)，接下來一行有n×(n−1)/2個正整數。

(30 分):n≤6
(70 分): 無限制


Output

第一行輸出能生成出該Δ的最小字典序的陣列，第二行輸出能生成出該Δ的最大字典序的陣列。


Sample Input 1 

3
3 4 7
Sample Output 1

0 3 7
0 4 7
Sample Input 2 

5
1 2 3 3 5 5 6 8 10 11
Sample Output 2

0 1 3 6 11
0 5 8 10 11


# 重組問題

## 題目想法

一開始看到這一題的時候，我其實有點害怕，因為這一題一旦距離變多，就算是自己拿紙筆慢慢畫圖，

也要花很多時間，而且還不一定能找出所有可能的組合，很容易就會有遺漏。

所以當下我就知道，這一題一定要找到一個能夠系統性搜尋所有可能的方法。

後來我查了不少資料，也問了很多問題，慢慢理解了這一題真正的解法。

等到理解整個思路之後，我也成功自己把程式完整寫了出來。

這一題我是利用 **DFS 搭配回溯（Backtracking）以及剪枝** 來完成搜尋。

另外，因為題目沒有特別限制距離的大小，所以我全部都使用 long long，避免之後因為資料範圍較大而發生整數溢位的問題。

## 解題過程

首先，我先建立題目所需要的變數，包含輸入的 n、儲存目前還沒有使用過距離的 multiset dist、

目前已經確定放入的點 points，以及最後要輸出的最小和最大字典序答案 `smallans` 和 `bigans`。

接著我建立了一個 addpoints() 函式，回傳型態為 bool，參數是一個 long long x，代表我要嘗試加入的新點。

我先建立一個 needdists，用來記錄如果加入這個點之後，它和目前所有已知點所形成的距離；

再建立一個 removed，記錄這一次暫時從 dist 中刪除了哪些距離，方便之後如果失敗可以完整復原。

接著遍歷目前的 points，把每一個點和 x 所形成的距離全部放進 needdists。

之後再依序檢查 needdists 中的每一個距離是否都存在於 dist 裡。

如果其中有任何一個距離不存在，就代表這個點不可能放在這個位置，因此直接把剛剛刪除的距離全部放回 dist，並回傳 false。

如果所有距離都存在，就代表目前這個點是合法的，因此把 x 加入 points，並回傳 true。

接著我再建立另一個 removedpoints() 函式，參數同樣是一個 long long x，代表剛剛加入的點。

這個函式會遍歷目前所有的點，把剛剛加入的點和其他點所形成的距離全部加回 dist，最後再把 points 裡面的這個點刪除，完成回溯。

之後就是整個程式最重要的 dfs()。

一開始先判斷 dist 是否已經完全被消耗掉。如果是，就代表目前所有的點都已經成功重建出來，

因此先把 points 複製一份並排序，之後更新目前字典序最小以及最大的答案，再直接返回上一層。

如果 dist 還沒有清空，就先取出目前剩餘距離中最大的距離 d。

根據這一題的性質，新加入的點只可能有兩種位置，一種是距離左端點 d 的位置，

另一種則是距離右端點 len-d 的位置，因此我只需要嘗試這兩種可能即可。

如果加入其中一個位置成功，就繼續往下一層 DFS 搜尋；等到搜尋完成之後，

再呼叫 removedpoints() 把剛剛加入的點以及刪除的距離全部恢復，繼續搜尋另一種可能。

最後進入 main()。

先輸入 n，並特別判斷 n==1 的情況。因為當 n 為 1 時，不存在任何兩兩距離，如果直接按照一般流程進行，

整個程式就無法正確運作，因此直接輸出兩行 0 後結束程式。

接著建立變數 m=n*(n-1)/2，方便表示距離的總數，再建立一個 `bt` 紀錄所有距離中的最大值。輸入所有距離時，同時更新 bt。

因為最大的距離一定就是最左端和最右端兩個點之間的距離，所以一開始就可以先把 0 和 bt 放進 points，並把這個距離從 dist 中刪除。

之後呼叫 dfs(bt) 開始搜尋所有可能的解，最後輸出 smallans 和 bigans 即可。

## 解題方法思考

我覺得這一題真正困難的地方，不是在寫程式，而是在理解整個搜尋的邏輯。

一開始我曾經想過是不是可以直接模擬，把每一個點慢慢放進去，但很快就發現這樣根本不可行。

因為隨著距離越來越多，要考慮的排列組合也會快速增加，不但很容易漏掉答案，效率也會變得非常差。

後來理解了題目的性質之後，我發現每一次都可以利用目前剩下的最大距離來推論新加入點的位置，

因此每一步其實只需要考慮兩種可能，再搭配 DFS、回溯以及剪枝，把不可能的情況提早排除，就可以有效地完成整個搜尋。

雖然這題的實作細節不少，但這種利用題目性質縮小搜尋範圍，再配合回溯完成搜尋的方法，也讓我對 DFS 和剪枝有了更深入的理解。

## 複雜度分析

**時間複雜度：**

** O(2ⁿ × n)**

每加入一個新的點，都可能需要嘗試兩種位置，不過每一次加入點時，
都會立即檢查它與所有已知點所形成的距離是否存在於 dist 中，
如果不符合就直接回溯，因此實際搜尋的節點數會因為剪枝而大幅減少。

**空間複雜度：**

**O(n²)。**

主要是 dist 儲存了所有兩兩距離，共有 n(n-1)/2 個元素，因此需要 O(n²) 的空間((n*n-n)/2->n*n);

另外 points、遞迴深度以及其他暫存陣列都只有 O(n)，因此整體空間複雜度仍然以 dist 為主。
