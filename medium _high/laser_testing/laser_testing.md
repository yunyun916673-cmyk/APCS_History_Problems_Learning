雷射測試
題目描述
一個雷射光從座標 (0,0) 向右邊發射，平面上有很多個鏡子。
問雷射光總共會反射幾次。
保證輸入資料中，雷射光不會形成無限循環。
鏡子使用三個數字表示：
(xi, yi, ti)
代表第 i 個鏡子的座標為：
(xi, yi)
其中：
ti = 0
代表鏡子擺放方式為：
/
斜線鏡。
ti = 1
代表鏡子擺放方式為：
\
反斜線鏡。
保證不會有兩個鏡子位於同一個位置。
輸入格式
第一行輸入一個正整數：
n
代表鏡子的數量。
接下來 n 行，每行有三個整數：
xi yi ti
代表第 i 個鏡子的位置與種類。
限制
1 <= n <= 250000
其中：
1 <= xi <= 30000
-30000 <= yi <= 30000

子題限制
20%
1 <= n <= 1000
1 <= xi <= 100
|yi| <= 100
80%
完整限制：
1 <= n <= 250000
1 <= xi <= 30000
|yi| <= 30000

輸出格式
輸出雷射光反射的次數。
範例
Input
5
2 0 0
2 2 1
4 2 0
4 4 1
6 4 0
Output
5


# 雷射測試

## 題目想法

一開始看到這個題目的時候，我第一個想到的是如果直接建立整張地圖來模擬雷射移動，那麼整個地圖大小可能會非常大。

雖然鏡子的數量只有 n 個，但是座標範圍可以到 30000，如果真的建立完整的二維陣列，不只會浪費大量空間，而且為了判斷雷射是否超出範圍，也會讓整個邏輯變得更加複雜。

因此我改變想法，不去建立整張地圖，而是只記錄鏡子的位置。

因為雷射只會在遇到鏡子的時候改變方向，所以其實我只需要快速找到目前方向上的下一個鏡子即可。

因此我建立兩個 unordered_map：

- rowsincols：利用 y 座標儲存同一條水平線上的所有 x 座標
- colsinrows：利用 x 座標儲存同一條垂直線上的所有 y 座標

這樣在雷射移動時，就可以快速找到下一個會遇到的鏡子。

另外，我使用 map<pair<ll,ll>,ll> 儲存每個鏡子的種類，原因是需要利用座標 (x,y) 作為 key，而 unordered_map 不能直接用 pair 作為 key，因此使用 map 來存取。

## 解題過程

首先建立題目要求的 n，並輸入所有鏡子的資料。

輸入每個鏡子時：

rowsincols[y].insert(x);
colsinrows[x].insert(y);
types[{x,y}]=t;
分別儲存：
該鏡子的 x 座標
該鏡子的 y 座標
該鏡子的種類
接著建立：
nowR
nowC
nowDIR
用來記錄雷射目前的位置以及方向。
一開始雷射從 (0,0) 出發，方向為向右。
另外建立 turn 代表雷射因為鏡子而轉向的次數。
方向設定如下：
0：向右
1：向下
2：向左
3：向上
接著建立 shoot() 函式模擬雷射移動。
函式內建立布林值 outal，代表雷射是否已經離開所有鏡子的範圍。
進入 while 迴圈後，根據目前方向分別處理：
向右
利用：
rowsincols[nowC].upper_bound(nowR)
找到同一條水平線上，右方最近的鏡子。
向左
利用：
rowsincols[nowC].lower_bound(nowR)
找到左方最近的鏡子。
向上
利用：
colsinrows[nowR].upper_bound(nowC)
找到上方最近的鏡子。
向下
利用：
colsinrows[nowR].lower_bound(nowC)
找到下方最近的鏡子。
如果沒有找到鏡子，代表雷射已經離開，因此將 outal 設為 true。
如果找到鏡子，則更新目前位置，並利用 types 判斷鏡子的種類。
根據：
/
\
決定下一次移動方向。
每遇到一次鏡子，就讓：
turn++;
最後當雷射離開後，輸出 turn。

##解題方法思考
這一題我覺得真正困難的地方並不是模擬雷射反射的邏輯，而是如何想到適合的資料結構。
一開始我也曾經想過直接建立地圖，但是後來發現座標範圍太大，不只需要大量空間，而且程式邏輯也會變得更加複雜。
後來想到雷射其實只會受到鏡子的影響，因此不需要知道整張地圖，只需要知道每個方向上最近的鏡子即可。
所以我利用兩個方向的資料儲存方式：
水平方向利用 y 找 x
垂直方向利用 x 找 y
再搭配 multiset 的排序特性，就可以快速找到下一個鏡子。
在實作過程中，我也遇到 iterator 使用上的問題。
原本我想使用 rend() 來判斷是否已經超過範圍，但是後來發現 rend() 是反向迭代器，和一般 iterator 的操作方式不同，因此會造成編譯錯誤。
後來改成使用 begin() 判斷，再確認 iterator 是否可以往前移動，成功解決問題。
複雜度分析
時間複雜度：
O(n log n)
原因：
首先輸入所有鏡子需要 O(n)。
模擬雷射時，每次遇到鏡子都需要使用 multiset 的 lower_bound() 或 upper_bound() 尋找下一個鏡子，單次操作為 O(log n)。
由於題目保證不會形成無限循環，因此最多只會經過所有鏡子，因此總時間複雜度為 O(n log n)。
空間複雜度：
O(n)
原因：
需要儲存所有鏡子的座標以及種類，每個鏡子會被存入相關資料結構中，因此空間複雜度為 O(n)。

##學習心得
我覺得這一題最大的收穫不是學會如何模擬雷射反射，而是學習如何避免建立不必要的資料。
一開始如果直接建立整張地圖，雖然概念上比較直覺，但是因為座標範圍很大，反而會讓問題變得更加困難。
後來發現其實不需要知道所有空格的資訊，只需要維護真正會影響結果的鏡子位置，就能大幅降低複雜度。
這也讓我了解到，遇到座標範圍很大的問題時，不一定要直接建立完整模型，而是要思考題目真正需要哪些資訊。
另外，這題也讓我更加熟悉 multiset 搭配 lower_bound() 和 upper_bound() 的使用方式，知道如何利用排序資料結構快速找到指定方向上的下一個位置。
