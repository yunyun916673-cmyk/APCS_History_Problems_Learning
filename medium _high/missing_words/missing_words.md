缺字問題
Description

給定一個大小為K個字母的集合和字串S，求出在使用該集合所組出長度為L字串中，不為字串S子字串的最小字典序字串為何。

例如字母集合{a,c,m}，其能組出長度為2的字串字典序由小到大排序為aa<ac<am<ca<cc<cm<ma<mc<mm。字串S為accaamcm，因此ma為不在S內的最小字典序字串。


Input

第一行為一個長度為K(1≤K≤10)的小寫字母字串代表字母集合，保證字元不重複且照字元由小到大排序。

第二行為一個正整數L(1≤L≤8,1≤KL≤6×105)。

第三行為小寫英文字串S(L≤|S|≤5×105)。

(20 分):|S|=1000
(80 分): 無限制


Output

輸出滿足題目要求的最小字典序字串


Sample Input 1 

acm
2
accaamcm
Sample Output 1

ma
Sample Input 2 

dp
3
dddppdpd
Sample Output 2

pdd


# 缺字問題

## 題目想法

一開始看到這一題的時候，我認為它主要是在利用 DFS 生成所有可能的字串組合。比起前一題連鎖反應需要處理比較複雜的搜尋狀態，這一題比較單純，只需要利用 DFS 將集合中的字元組合成長度為 L 的所有字串。

我先思考題目需要維護哪些資訊，發現需要知道字母集合 U、目標長度 L，以及原本的字串 S。

除此之外，我需要知道字串 S 中已經存在的所有長度為 L 的子字串，之後才能判斷 DFS 生成出的字串是否已經出現過。

因此我的想法是先利用 DFS 生成所有可能的字串，並且另外建立集合儲存 S 中出現過的字串，最後依照字典序找出第一個不存在的字串。

## 解題過程

首先建立字串 U、S，以及長度 L。

接著建立兩個集合：

- all：儲存所有利用集合 U 可以組出的長度為 L 的字串。
- every：儲存字串 S 中所有連續長度為 L 的子字串。

之後建立 DFS 函式，參數使用目前組合出的字串 `cur`。

在 DFS 中，首先判斷目前字串長度是否已經達到 L。

如果達到 L，代表已經完成一個可能答案，因此將它加入 `all`。

如果還沒有達到 L，則遍歷集合 U 中的每一個字元，將字元加入目前字串後繼續 DFS。

完成 DFS 後，`all` 中就會包含所有可能組合。

接著利用迴圈掃描字串 S，將每一個長度為 L 的連續子字串加入 `every`。

最後遍歷 `all`，判斷目前字串是否存在於 `every` 中。

如果不存在，代表這個字串就是題目要求的最小字典序缺少字串，因此直接輸出並結束程式。

## 解題方法思考

這題最主要的想法是利用 DFS 生成所有可能狀態。

一開始我想到的方法是先把所有可能的字串全部建立出來，再逐一判斷是否存在於 S 中。

因為題目限制：

K^L ≤ 600000

所以所有可能字串數量並不會太大，可以接受使用 DFS 生成。

另外，我也發現因為 DFS 遍歷字元時是按照集合 U 的順序，而 U 本身已經按照字典序排序，因此生成出的字串順序本身就是字典序。

所以其實不一定需要額外建立 `all` 集合，也可以在 DFS 生成字串時直接判斷是否存在於 `every`，找到第一個不存在的字串後立即輸出。

這讓我了解到 DFS 不一定只能用來列出所有答案，也可以直接利用搜尋順序找到需要的答案。

## 複雜度分析

時間複雜度：  O(K^L*L + S*L)

DFS 需要生成 K^L 個長度為 L 的字串，每個字串建立需要 O(L) 時間。

另外需要掃描 S 中所有長度為 L 的子字串，每次取出子字串需要 O(L) 時間。

空間複雜度：  O(K^L*L)

需要儲存所有可能組合的字串，以及 S 中出現過的子字串。

## 學習心得

這一題讓我練習了利用 DFS 生成所有可能組合的方法。

在做過不少 APCS 歷屆試題後，我發現如果題目和字串有關，通常會比一些需要使用其他資料結構的題目單純一些，因為字串問題的狀態變化通常比較固定。

不過這題也讓我發現，DFS 並不只是單純把所有答案列出來。有時候利用 DFS 的搜尋順序，就可以直接找到符合條件的第一個答案，不需要額外儲存全部結果。

原本我的做法是建立 all 集合保存所有可能字串，但後來思考後發現，其實可以在生成過程中直接判斷，減少不必要的空間使用。

這讓我對 DFS 的應用方式有更多理解，也開始注意到解法中是否有可以簡化或優化的地方。
