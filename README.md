# APCS 歷屆題目學習紀錄

## 一、專案動機

為了提升程式設計能力，準備 APCS 檢測，故而整理歷屆試題的解題過程與學習紀錄。

過程中，我發現要解決問題，最重要的是分析問題、選擇合適的方法，並經由測試和修正找到更好的解法。故此，我建立該專案，記錄自己關於APCS歷屆試題的學習歷程。

## 二、學習目標

- C++ 程式設計與常用語法

- 資料結構與演算法基礎

- 分析問題與設計解法

- APCS 歷屆題目練習，累積實際解題經驗

## 三、學習方法

每完成一道題目，會整理出：

- 1. 題目的核心問題


- 2. 解題思考過程


- 3. 使用的資料結構與演算法


- 4. 程式實作方式


- 5. 遇到的錯誤與改進方法

藉由整理解題紀錄，回顧自己在思考方式上的改變，了解哪些地方需要加強。

## 四、專案整理方式

依照題目難度進行分類：

- 簡單題：基本語法與基礎解題技巧

- 中等題：常見演算法與資料結構應用

- 中高難度題：較複雜之思考與優化

- 困難題：進階演算法與解題策略


每一道題目皆包含程式碼以及個人的解題紀錄。

## 五、學習成果與反思

透過持續練習 APCS 歷屆題目，我的思考方式逐漸轉變為如何分析問題、降低複雜度，以及選擇更適合的解法。
在遇到無法解出的題目時，我會藉由測試、查尋資料與重設方法來修正錯誤，累積解決問題的經驗。

## 六、未來規劃

希望提升演算法能力，並將程式設計能力延伸至多元領域，如人工智慧、資訊安全與實際專案開發等。
